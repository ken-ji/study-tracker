# 個人用学習管理アプリ 要件定義

## 概要

個人の学習進捗を管理・可視化するWebアプリ。インターネット接続なしでも利用でき、スマートフォン・PCどちらでも快適に使える。

---

## 技術要件

| 項目 | 内容 |
|------|------|
| ホスティング | GitHub Pages（静的ファイルのみ） |
| バックエンド | なし（サーバーレス） |
| データ保存 | LocalStorage |
| オフライン対応 | Service Worker + PWA |
| レスポンシブ | スマホファースト設計 |
| フレームワーク | Vanilla JS（または軽量ライブラリ） |

---

## 機能要件

### 1. 学習記録（コア機能）
- [ ] 科目・カテゴリの登録・編集・削除
- [ ] 学習セッションの記録（科目、日時、学習時間、メモ）
- [ ] 学習時間のタイマー機能（開始・停止・リセット）
- [ ] 過去の記録の一覧表示・編集・削除

### 2. 目標管理
- [ ] 科目ごとの目標時間設定（日次・週次・月次）
- [ ] 目標に対する進捗表示（プログレスバー）
- [ ] 目標達成時の通知・バッジ表示

### 3. ダッシュボード
- [ ] 今日の学習時間サマリー
- [ ] 今週・今月の累計時間
- [ ] 科目別の学習時間グラフ（棒グラフ / 円グラフ）
- [ ] 学習連続日数（ストリーク）表示
- [ ] カレンダーヒートマップ（学習量の可視化）

### 4. データ管理
- [ ] データのエクスポート（JSON形式）
- [ ] データのインポート（バックアップ復元）
- [ ] データ全削除（確認ダイアログ付き）

### 5. PWA対応
- [ ] ホーム画面への追加（インストール）
- [ ] オフライン時でも全機能利用可能
- [ ] アプリアイコン・スプラッシュ画面
- [ ] プッシュ通知（学習リマインダー）※任意

---

## 非機能要件

### パフォーマンス
- 初回ロード：3秒以内（低速回線含む）
- LocalStorage 操作：即時反応（UI ブロックなし）
- LocalStorage 使用量：5MB 以内を目安に警告表示

### UX / デザイン
- スマホ画面（375px〜）でタップ操作しやすいUI
- ダークモード対応
- アニメーションは控えめ（バッテリー・パフォーマンス配慮）
- 日本語 UI

### データ永続性
- LocalStorage はブラウザ・デバイスごとに独立
- エクスポート/インポートで手動バックアップを推奨する旨を明示

### ブラウザ対応
- Chrome / Safari（iOS）/ Firefox 最新版
- iOS Safari の LocalStorage 制限（プライベートモード）を考慮

## 開発スコープ

### MVP（最初にリリースするもの）
1. 科目登録・編集
2. 学習セッション手動記録
3. タイマー機能
4. ダッシュボード（今日・今週の集計、科目別グラフ）
5. PWA 化（オフライン対応 + ホーム追加）

### Phase 2（MVP 後に追加）
- 目標管理・進捗バー
- カレンダーヒートマップ
- ストリーク表示
- ダークモード
- データ export/import

### 対象外（スコープ外）
- ユーザー認証・クラウド同期
- 複数デバイス間のリアルタイム同期
- 他ユーザーとの共有機能

## 制約・注意事項

- GitHub Pages は静的ファイルのみ。サーバーサイド処理は不可
- LocalStorage はオリジン（ドメイン）単位で保存される
- iOS Safari のプライベートブラウズでは LocalStorage が制限される場合あり
- Service Worker の更新タイミングに注意（キャッシュ戦略の設計が重要）
